MÉTHODE DU PREMIER HARMONIQUE

TP N°3 - Saturation & Cycle Limite (Systèmes Non Linéaires)

1. Analyse Théorique & Calculs (Q1, Q10, Q11)

Gain Équivalent (Fonction Descriptive) : La méthode du premier harmonique postule que si un système linéaire agit comme un filtre passe-bas, il filtre les harmoniques supérieures créées par la non-linéarité. La non-linéarité est ici une saturation (seuil h = 1.5, gain k = 2). La limite linéaire est es = h/k = 0.75.

Si a ≤ 0.75 : N(a) = k = 2 (Comportement linéaire)
Si a > 0.75 : N(a) = (4/π) * [ arcsin(0.75/a) + (0.75/a)*√(1 - (0.75/a)²) ]

Calcul de la Pulsation Critique (ωc) : La fonction de transfert du système linéaire (donnée dans le TP) est :

H(s) = 5 / (0.001 s³ + 0.03 s² + 0.3 s + 1)
H(jω) = 5 / [ (1 - 0.03ω²) + j(0.3ω - 0.001ω³) ]

L'intersection du lieu de Nyquist avec l'axe réel négatif (nécessaire pour la condition d'auto-oscillation de Barkhausen) se produit lorsque la partie imaginaire s'annule :

Im(H) = 0 => 0.3ω - 0.001ω³ = 0
ω² = 300 => ωc = √300 ≈ 17.32 rad/s

Calcul de l'Amplitude du Cycle Limite (ac) : En remplaçant ωc dans la partie réelle, on trouve le point exact où le système coupe l'axe réel :

Re(H(jωc)) = 5 / (1 - 0.03*300) = 5 / (1 - 9) = -0.625
Condition du cycle limite : H(jωc) = -1 / N(ac)
-1 / N(ac) = -0.625 => N(ac) = 1.6

En résolvant l'équation transcendante N(ac) = 1.6 numériquement, on trouve ac ≈ 1.08. C'est l'amplitude théorique des oscillations que nous allons vérifier par la simulation RK4.

2. Plan Complexe : Nyquist & Lieu Critique

Le tracé bleu représente la réponse fréquentielle du système linéaire H(jω). La courbe pointillée rouge représente l'inverse changé de signe du gain équivalent -1/N(a). Le point de croisement confirme mathématiquement l'existence d'un cycle limite.

% =========================================================================
% MODULE: Systèmes Non Linéaires (U.S.T.H.B) - TP N°3
% OBJET : Illustration de la méthode du premier harmonique avec 
%         une non-linéarité de type Saturation.
% =========================================================================

function tp3_snl_resolution_complete()
    % ---------------------------------------------------------------------
    % Q1: Définition des paramètres physiques de la saturation.
    % Le gain linéaire 'k' est la pente à l'origine.
    % La limite 'h' est la valeur maximale en sortie.
    % 'es' représente l'entrée seuil à partir de laquelle le système sature.
    % ---------------------------------------------------------------------
    h = 1.5; 
    k = 2; 
    es = h/k; % es = 0.75
    
    % ---------------------------------------------------------------------
    % Q2: Représentation du gain équivalent N(a)
    % L'amplitude 'a' balaie l'intervalle [0, 25] avec un pas de 0.1.
    % On utilise arrayfun pour appliquer notre fonction personnalisée 
    % eval_Na à chaque élément du vecteur de manière vectorisée.
    % ---------------------------------------------------------------------
    a_vec = 0:0.1:25;
    Na_vec = arrayfun(@(a) eval_Na(a, k, es), a_vec);
    
    figure('Name', 'Gain Equivalent');
    plot(a_vec, Na_vec, 'LineWidth', 2); grid on;
    title('Q2: Gain équivalent N(a) pour la Saturation'); 
    xlabel('Amplitude de l''oscillation (a)'); ylabel('N(a)');

    % ---------------------------------------------------------------------
    % Q3 & Q4: Intégration des équations différentielles (Système Bouclé)
    % Le système est donné sous forme d'état: dx/dt = Ax + Bu.
    % L'entrée 'u' est la sortie de la non-linéarité avec un signe négatif 
    % dû à la boucle de contre-réaction : u = -sat(x1).
    % On utilise ode45, basé sur la méthode de Runge-Kutta explicite (4,5),
    % qui gère automatiquement le pas d'intégration pour limiter l'erreur.
    % ---------------------------------------------------------------------
    tspan = [0 20]; % Intervalle d'intégration temporel t ∈ [0, 20]
    x0 = [0.1; 0; 0]; % Condition initiale de l'état du système
    
    % Définition matricielle dans une fonction anonyme
    sys_cl = @(t, x) [
        x(2);
        x(3);
        -1000*x(1) - 300*x(2) - 30*x(3) + 5000*(-sat_func(x(1), k, h))
    ];

    [t, x_out] = ode45(sys_cl, tspan, x0);

    % ---------------------------------------------------------------------
    % Q5 & Q6: Tracé de la sortie temporelle x1(t)
    % Remarque: On observe un régime transitoire suivi de l'établissement 
    % d'un régime oscillatoire permanent. C'est le Cycle Limite.
    % ---------------------------------------------------------------------
    figure('Name', 'Simulation Temporelle');
    plot(t, x_out(:,1), 'LineWidth', 1.5); grid on;
    title('Q5: Évolution de la sortie du système x_1(t)'); 
    xlabel('Temps (secondes)'); ylabel('Amplitude x_1(t)');

    % ---------------------------------------------------------------------
    % Q7 & Q8: Diagramme de Nyquist et Lieu Critique dans le Plan Complexe
    % H(s) est la fonction de transfert de la partie linéaire. On évalue
    % H(jw) en substituant s par jw. 
    % Le lieu critique correspond à l'inverse négatif du gain équivalent.
    % ---------------------------------------------------------------------
    w = linspace(0, 100, 10000);
    s = 1i * w;
    H_jw = 5 ./ (0.001*s.^3 + 0.03*s.^2 + 0.3*s + 1);

    figure('Name', 'Plan Complexe de Nyquist');
    plot(real(H_jw), imag(H_jw), 'b', 'LineWidth', 1.5); hold on; grid on;
    
    % Tracé du lieu -1/N(a). Il se situe entièrement sur l'axe réel négatif 
    % car la saturation est une non-linéarité univoque (sans mémoire).
    lieu_critique = -1 ./ Na_vec;
    plot(lieu_critique, zeros(size(lieu_critique)), 'r--', 'LineWidth', 2);
    title('Q7-Q8: Intersection de Nyquist et du Lieu Critique');
    legend('H(j\omega)', '-1/N(a)');
    axis([-1 0.2 -3 1]);

    % ---------------------------------------------------------------------
    % Q9, Q10, Q11: Calcul du point d'intersection exact (Cycle Limite)
    % Le point de croisement vérifie l'équation de l'auto-oscillation.
    % L'équation transcendante N(a_c) = 1.6 est résolue numériquement avec
    % la fonction MATLAB 'fzero', qui cherche la racine d'une fonction.
    % ---------------------------------------------------------------------
    f_opt = @(a) eval_Na(a, k, es) - 1.6;
    ac_num = fzero(f_opt, 1); % On donne 1 comme point de départ (guess)
    fprintf('Amplitude théorique du cycle limite ac = %.4f\n', ac_num);
end

% =========================================================================
% SOUS-FONCTIONS AUXILIAIRES
% =========================================================================

% Évaluation mathématique du gain équivalent N(a) pour une saturation pure
function N = eval_Na(a, k, es)
    if a <= es
        % Zone totalement linéaire, le gain est constant
        N = k;
    else
        % Application de la formule intégrale de Fourier (fondamental)
        alpha = es / a;
        N = (4*k/pi) * (asin(alpha) + alpha * sqrt(1 - alpha^2)) / 2; 
    end
end

% Modélisation physique du bloc de saturation
function y = sat_func(x, k, h)
    if x > h/k
        y = h;       % Écrêtage positif
    elseif x < -h/k
        y = -h;      % Écrêtage négatif
    else
        y = k * x;   % Régime linéaire proportionnel
    end
end
